# jaree-client-repo 아키텍처

> 프론트엔드 클라이언트 애플리케이션의 아키텍처 설계 문서

## 개요

jaree-client-repo는 React 기반의 단일 페이지 애플리케이션(SPA)으로 설계되었습니다. Vite를 빌드 도구로 사용하여 빠른 개발 경험과 최적화된 프로덕션 빌드를 제공합니다.

## 주요 컴포넌트

### 1. 진입점 (main.tsx)
- React 애플리케이션의 진입점
- ReactDOM을 사용하여 앱을 DOM에 마운트

### 2. 메인 앱 컴포넌트 (App.tsx)
- 애플리케이션의 루트 컴포넌트
- 전역 레이아웃 및 라우팅 관리

### 3. 컴포넌트 구조
- `components/`: 재사용 가능한 UI 컴포넌트
  - `EnvInfo.tsx`: 환경 변수 정보 표시 컴포넌트

## 데이터 흐름

### 환경 변수 관리
1. 환경 변수는 `env/` 디렉토리의 `.env` 파일에서 관리
2. Vite는 `VITE_` 접두사로 시작하는 변수만 클라이언트에 노출
3. `import.meta.env`를 통해 런타임에 접근 가능

### API 통신
- 환경 변수 `VITE_API_BASE_URL`을 통해 API 서버와 통신
- 향후 API 클라이언트 라이브러리 통합 예정

## 기술 결정사항

### Vite 선택 이유
- **빠른 개발 서버**: HMR(Hot Module Replacement) 지원
- **빠른 빌드**: esbuild 기반의 빠른 번들링
- **TypeScript 지원**: 기본 TypeScript 지원
- **플러그인 생태계**: 풍부한 플러그인 지원

### React 19 선택 이유
- **최신 기능**: 최신 React 기능 활용
- **성능 개선**: 향상된 렌더링 성능
- **개발자 경험**: 개선된 개발자 도구

### Tailwind CSS 선택 이유
- **유틸리티 우선**: 빠른 스타일링
- **반응형 디자인**: 기본 반응형 유틸리티
- **커스터마이징**: 쉬운 테마 커스터마이징

## 빌드 프로세스

### 개발 모드
1. Vite 개발 서버 시작
2. TypeScript 타입 체크 (tsc)
3. HMR을 통한 실시간 업데이트

### 프로덕션 빌드
1. TypeScript 컴파일 (tsc -b)
2. Vite 프로덕션 빌드
3. 코드 최적화 및 트리 쉐이킹
4. `dist/` 디렉토리에 결과물 생성

## 환경별 설정

### 개발 환경 (dev)
- 소스맵 생성
- 개발 도구 활성화
- 상세한 에러 메시지

### 프로덕션 환경 (production)
- 코드 최소화
- 소스맵 제외 (선택적)
- 성능 최적화

## 보안 고려사항

- 환경 변수는 `VITE_` 접두사로 시작하는 것만 클라이언트에 노출
- 민감한 정보는 서버 사이드에서 관리
- API 키 등은 클라이언트에 노출하지 않음
